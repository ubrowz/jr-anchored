
What this script does:
       Answers the question: "Can we trust our measurement tool?"

       Before using a gauge to make decisions about your product, you need to
       know whether the variation you see in the numbers comes from real
       differences between parts, or from the instrument itself being noisy.

       The experiment: take a fixed set of parts, have multiple operators each
       measure every part multiple times. The script then splits the observed
       variation into three sources:

         Repeatability — same operator, same part, different trials.
                         How consistent is the gauge with itself?
         Reproducibility — same part, different operators.
                           Do different people get the same reading?
         Part-to-Part — the real signal you care about.

       The key output is %GRR: what percentage of total observed variation is
       due to the measurement system. Industry rule of thumb:
         < 10%   Acceptable
         10-30%  Marginal — evaluate based on application
         > 30%   Unacceptable — gauge must be improved before use

       A second output, ndc (number of distinct categories), tells you how
       many groups the gauge can reliably distinguish. ndc >= 5 is required.

Description: Gauge R&R analysis using the two-way ANOVA method. Computes
             repeatability, reproducibility, part-to-part variation, %GRR,
             and ndc from a balanced part/operator/value dataset.
             Saves a four-panel PNG to ~/Downloads/.

Usage: jrc_msa_gauge_rr <data.csv> [--tolerance <value>]

Arguments:
       data.csv             CSV file with columns: part, operator, value
                            All part-operator combinations must have the same
                            number of replicates (balanced design).
       --tolerance <value>  Optional. Process tolerance (USL - LSL). When
                            supplied, %GRR vs tolerance is also reported.
       -h | --help          Show this help message

Input CSV format:
       part,operator,value
       1,A,10.12
       1,A,10.15
       1,B,10.11
       ...

       Minimum: 2 parts, 2 operators, 2 replicates per cell.

Output — Terminal:
       ANOVA table (part, operator, part:operator, residual)
       Variance components with % contribution
       Study variation (%Study Var) for each component
       %GRR vs tolerance (if --tolerance supplied)
       Number of distinct categories (ndc)
       Pass/fail verdict

Output — PNG (~/Downloads/):
       Four-panel chart:
         Top-left:  Components of variation bar chart
         Top-right: Measurements by part (box + dot)
         Bot-left:  Measurements by operator (box + dot)
         Bot-right: Part x Operator interaction plot

Acceptance criteria (industry standard):
       %GRR < 10%   Acceptable
       %GRR 10-30%  Marginal — evaluate based on application
       %GRR > 30%   Unacceptable
       ndc >= 5     Acceptable

Examples:
       jrc_msa_gauge_rr data.csv
       jrc_msa_gauge_rr data.csv --tolerance 0.5
