
What this script does:
       Attribute Agreement Analysis is Gauge R&R for inspectors making
       categorical judgments — pass/fail, go/no-go, defect ratings — instead
       of continuous measurements.

       Three questions it answers:

         Within-appraiser (repeatability):
           Does each inspector agree with themselves? Show the same part twice
           without them knowing and see if they give the same answer both
           times. An inspector who flip-flops on the same part is unreliable.

         Between-appraiser (reproducibility):
           Do all inspectors agree with each other on the same parts? One
           inspector saying PASS and another saying FAIL on the same part
           means the inspection process is inconsistent.

         Each appraiser vs Reference (accuracy):
           If you know the correct answer for each part (from an expert or
           reference standard), how often does each inspector get it right?
           An inspector can be consistent with themselves but still
           consistently wrong.

       The key statistic is Kappa. It measures agreement beyond what you
       would expect by chance. A Kappa of 1.0 means perfect agreement.
       A Kappa of 0 means the inspector is essentially guessing. Negative
       Kappa means they are doing worse than chance.

         Kappa > 0.90   Acceptable
         Kappa 0.70-0.90 Marginal
         Kappa < 0.70   Unacceptable

       Within-appraiser and each-vs-reference use Cohen's Kappa (two raters).
       Between-appraiser uses Fleiss' Kappa (multiple raters simultaneously).

Description: Attribute Agreement Analysis. Computes within-appraiser
             repeatability, between-appraiser reproducibility (Fleiss' Kappa),
             and — when a reference column is provided — accuracy vs the
             correct answer (Cohen's Kappa per appraiser). Saves a two-panel
             PNG (% agreement bars + Kappa chart) to ~/Downloads/.

Usage: jrc_msa_attribute <data.csv>

Arguments:
       data.csv    CSV with columns: part, appraiser, trial, rating.
                   Optional column 'reference': the correct answer for each
                   part. Must be the same value for all rows of a given part.
                   Balanced design required: same number of trials for every
                   part-appraiser combination.
       -h | --help Show this help message

Input CSV format (without reference):
       part,appraiser,trial,rating
       1,A,1,PASS
       1,A,2,PASS
       1,B,1,PASS
       1,B,2,FAIL
       ...

Input CSV format (with reference):
       part,appraiser,trial,rating,reference
       1,A,1,PASS,PASS
       1,A,2,PASS,PASS
       1,B,1,PASS,PASS
       ...

       Minimum: 2 appraisers, 2 parts, 2 trials per appraiser-part combination.
       Recommended: 3 appraisers, 30 parts, 2-3 trials.

Output — Terminal:
       Within-appraiser table (%Within, Kappa, verdict per appraiser)
       Between-appraiser summary (%agree, Fleiss' Kappa, verdict)
       Each appraiser vs reference (if reference provided)
       Overall verdict

Output — PNG (~/Downloads/):
       Left panel:  % Agreement bar chart (within per appraiser, vs reference)
       Right panel: Kappa values for all comparisons

Acceptance criteria (industry standard):
       Kappa > 0.90    Acceptable
       Kappa 0.70-0.90 Marginal
       Kappa < 0.70    Unacceptable
       % Agreement > 90%  Acceptable
       % Agreement 80-90% Marginal
       % Agreement < 80%  Unacceptable

Example:
       jrc_msa_attribute inspection_data.csv
