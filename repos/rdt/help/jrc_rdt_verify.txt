
Description: Evaluates whether a pre-specified reliability claim is demonstrated by
             actual RDT test results. Reads a CSV of unit test times and pass/fail
             statuses. Reports Binomial (Clopper-Pearson) and Weibayes verdicts
             (PASS/FAIL) against the claim. Saves a two-panel PNG to ~/Downloads/.
             Use after jrc_rdt_plan once testing is complete.

Usage: jrc_rdt_verify <data.csv> --reliability R --confidence C --target_life T [options]

Arguments:
	data.csv            Path to CSV file with test results (see CSV format below).
	--reliability R     Reliability claimed in the test plan (0 < R < 1), e.g. 0.95.
	                    Must match the value used in jrc_rdt_plan.
	--confidence C      Confidence level from the test plan (0 < C < 1), e.g. 0.90.
	                    Must match the value used in jrc_rdt_plan.
	--target_life T     Life at which reliability was claimed, e.g. 5000.
	                    Must match the value used in jrc_rdt_plan.
	--time_col NAME     Name of the column containing test times (default: "time").
	--status_col NAME   Name of the column containing event indicator (default: "status").
	                    0 = survived to end of test (right-censored).
	                    1 = failed at the recorded time.
	--beta B            Weibull shape parameter. Enables Weibayes evaluation alongside
	                    the Binomial result. Must match the value used in jrc_rdt_plan.
	                    Obtain from jrc_weibull, or use the same assumed value as the plan.
	--accel_factor AF   Life-extension multiplier used in the test (default: 1.0).
	                    t_eff = time * accel_factor for each unit.
	                    Must match the value used in jrc_rdt_plan.
	-h | --help         Show this help message.

CSV format:
	The CSV must contain at least a time column and a status column (default names:
	"time" and "status"). A row identifier column is recommended but not required.

	  unit_id,time,status
	  1,5000,0
	  2,4850,1
	  3,5000,0
	  ...

	status = 0: the unit survived to the recorded time (test ended, unit still alive).
	status = 1: the unit failed at the recorded time.

	Units that failed AFTER target_life (t_eff > target_life) are treated as
	suspensions for the claim — they passed the required life before eventually failing.

Methods:
	Binomial / Clopper-Pearson (always reported):
	  Counts units with t_eff <= target_life AND status=1 as failures (k).
	  All other units count as suspensions (n-k).
	  Computes the exact upper confidence bound on the failure fraction:

	    F_upper = qbeta(C, k+1, n-k)     [Clopper-Pearson exact]
	    R_lower = 1 - F_upper
	    PASS if R_lower >= reliability_claim

	Weibayes (--beta required):
	  Uses accumulated Weibull time from ALL units. Failures at or before target_life
	  count toward k; all units contribute their actual effective time to T*.

	    T* = sum( t_eff_i ^ beta )   over all n units
	    R_demo = exp( -target_life^beta * qchisq(C, 2*(k+1)) / (2 * T*) )
	    PASS if R_demo >= reliability_claim

	  Equivalently: PASS iff T* >= T_threshold, the same criterion as jrc_rdt_plan.

	The Overall Verdict is PASS if either Binomial or Weibayes is PASS. When the
	two methods disagree, a note is printed; the Weibayes result is more powerful
	when beta is well-supported by prior data.

Output — Terminal:
	Summary: n, k, suspensions, units failed beyond target life.
	Binomial section: F_upper, R_lower, margin, verdict.
	Weibayes section (if --beta): T*, T_threshold, T*/T_threshold ratio, eta, R_demo, margin, verdict.
	Overall Verdict: PASS or FAIL.

Output — PNG:
	Saved to ~/Downloads/ with timestamp prefix YYYYMMDD_HHMMSS_jrc_rdt_verify.png.
	Panel 1: timeline — each unit as a horizontal line to t_eff; failures marked
	         with X (red); survivors with + (blue); dashed vertical line at target_life.
	Panel 2: demonstrated reliability bars — Claim, Binomial R_lower, Weibayes R_demo
	         (if applicable). Green = PASS, red = FAIL. Claim shown as dashed line.

Notes:
	The Binomial and Weibayes methods are mathematically related but not identical.
	At k=0 and accel_factor=1 they give the same R_lower/R_demo. At k>0 or
	accel_factor>1, Weibayes benefits from units tested beyond target_life while
	Binomial does not.

	Failures beyond target_life are treated as suspensions in BOTH methods:
	  - Binomial k: only counts failures at or before target_life.
	  - Weibayes T*: includes the unit's full accumulated t_eff (which > target_life),
	    contributing more than target_life^beta to T*. This is slightly conservative.

	The accel_factor and beta values MUST match those used in jrc_rdt_plan. Using
	different values is statistically invalid and will produce incorrect verdicts.

	This script exits 0 for both PASS and FAIL. Non-zero exit means an input or
	runtime error. Check the error message for the cause.

FDA/regulatory context:
	The test plan (jrc_rdt_plan) and this verification script (jrc_rdt_verify) together
	constitute the pre-specified statistical test protocol required under 21 CFR 820.30(f)
	and ISO 13485:2016 section 7.3.6. The plan must be pre-specified (written before
	testing begins). The acceptance criteria (R, C, k_allowed, beta, accel_factor) must
	not be changed after data collection starts.

Examples:
	Basic verification — 45 units all survived, Bogey mode:
	  jrc_rdt_verify results.csv --reliability 0.95 --confidence 0.90 --target_life 5000
	  Result: PASS (R_lower = 0.9501 >= 0.95).

	With Weibayes — same data, beta = 2.0:
	  jrc_rdt_verify results.csv --reliability 0.95 --confidence 0.90 --target_life 5000 --beta 2.0
	  Result: Both Binomial and Weibayes PASS.

	With life extension — test to 2x target_life, beta = 2.0:
	  jrc_rdt_verify results.csv --reliability 0.95 --confidence 0.90 --target_life 5000 --beta 2.0 --accel_factor 2.0
	  Result: Each unit's t_eff = time * 2; T* benefits from the extended test.

	Custom column names:
	  jrc_rdt_verify data.csv --reliability 0.95 --confidence 0.90 --target_life 5000 --time_col hours --status_col event
